      30             :  * destroyed.
      31             :  *
      32             :  * The general idea is that opal_list_item_t objects can be put on an
      33             :  * opal_list_t.  Hence, you create a new type that derives from
      34             :  * opal_list_item_t; this new type can then be used with opal_list_t
      35             :  * containers.
      36             :  *
      37             :  * NOTE: opal_list_item_t instances can only be on \em one list at a
      38             :  * time.  Specifically, if you add an opal_list_item_t to one list,
      39             :  * and then add it to another list (without first removing it from the
      40             :  * first list), you will effectively be hosing the first list.  You
      41             :  * have been warned.
      42             :  *
      43             :  * If OPAL_ENABLE_DEBUG is true, a bunch of checks occur, including
      44             :  * some spot checks for a debugging reference count in an attempt to
      45             :  * ensure that an opal_list_item_t is only one *one* list at a time.
      46             :  * Given the highly concurrent nature of this class, these spot checks
      47             :  * cannot guarantee that an item is only one list at a time.
      48             :  * Specifically, since it is a desirable attribute of this class to

      51             :  *
